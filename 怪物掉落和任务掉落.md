# 挂机面板明雷怪物掉落 与 任务奖励 说明

> 本文档针对"挂机面板"（`Script/系统处理类/辅助内挂类.lua`）里"明雷表"中可勾选自动挂机的场景明雷内容（如捣乱年兽、夏日炎炎等），整理其怪物构成、掉落道具与难度分级；并附任务/副本战斗奖励的配置说明。

## 一、数据来源与字段说明

- **挂机面板**：客户端协议550下发数据，由 `辅助内挂类:初始化明雷表()`（`Script/系统处理类/辅助内挂类.lua` 164-229行）拼出"明雷表"，玩家勾选后由 `挂机定时器()`（389行起）定时刷怪、自动挂机清怪。
- **掉落/奖励数值来源**：按活动中文名为 key 的 `HDPZ` 表，默认值写在 `Script/数据中心/活动配置.lua`，**运行时以根目录 `全局bl.txt`（GBK编码序列化缓存）中的实际值为准**（本文档全部数据取自该缓存文件的实际内容）。
- **结算逻辑**：`Script/任务处理类/自写活动.lua` 中各活动的"完成XXX(id组)"函数在战斗胜利后调用，按 `HDPZ[活动名].爆率`（百分制，掷骰≤该值即中奖）判定是否触发一次随机产出，中奖后调用 `糖果产出物品计算(ITEM)` 从 `ITEM` 权重表按权重随机抽取1个 key，再交给 `糖果生成产出(key,hd)` 分发（`Script/数据中心/活动配置.lua` 3-30行）。
- **字段含义**：

  | 字段                   | 含义                                                                  |
  | -------------------- | ------------------------------------------------------------------- |
  | 排序                   | 活动在明雷表中的顺位，数值越大通常对应角色等级越高/越晚期开放的内容，可作为难度参考                          |
  | 经验                   | 经验倍率，结算经验 = 平均等级 × 活动基础经验随机值 × 该倍率（无条件发放，与爆率无关）                                  |
  | 银子                   | 银子倍率，结算银子 = (平均等级×160+15000) × 该倍率（无条件发放，与爆率无关）                                  |
  | 爆率                   | 触发"随机产出一次道具"的概率（百分制）；未命中则本次结算只有经验/银子，没有任何道具                                    |
  | 书铁/宝石/怪物卡片/金丹/宝珠钨金珍珠 | **不是必发的道具或材料数量**，只是5个"备用参数"，仅当 ITEM 权重池里恰好配了 `装备书铁`/`灵饰书铁`/`宝宝书铁`/`宝石`/`九转金丹`/`怪物卡片`/`钨金`/`附魔宝珠`/`珍珠` 这类特定 key 且被权重抽中时，才会读取对应字段的区间来决定实际道具的等级/数量；如果活动的 ITEM 池里根本没放这些 key（本文档统计的绝大多数明雷活动都是这种情况），这5个字段就是**完全不会生效的死配置**，参见下方"六、书铁等材料为何不掉落"专节 |
  | ITEM                 | 中奖后按权重随机抽取1个 key 的道具池，数字为权重（非概率百分比）；多数 key 直接就是道具名（如"金柳露"），少数特殊 key（见上一行）会触发`__Funfa`表里的转换函数，实际发放的道具名跟 key 本身不同 |
  | 难度                   | 缓存中该字段目前固定为1（仅"自动抓鬼"为2），**不是有效的难度分级依据**，本文档改用"星级/段位后缀"和"排序顺位"作为难度参考 |

## 二、完整样例：怪物构成 + 掉落 + 任务奖励

### 1. 捣乱年兽

- 场景：地图1501、1193（各刷20只），任务类型1515，战斗类型160030
- 每日次数上限：50次
- **怪物构成**（一场战斗共10只，等级=队伍最高等级，`怪物属性:捣乱年兽()`）：

  | 怪物   | 模型   | 数量  | 定位  |
  | ---- | ---- | --- | --- |
  | 捣乱年兽 | 古代瑞兽 | 1   | 首领  |
  | 赤水蛟龙 | 蛟龙   | 2   | 小怪  |
  | 碧玉王八 | 龙龟   | 2   | 小怪  |
  | 闹事妖精 | 兔子怪  | 5   | 小怪  |

- **掉落**（HDPZ["捣乱年兽"]：经验倍率4，银子倍率1，爆率70）
  - ITEM权重池：金柳露10、高级召唤兽内丹10、高级魔兽要诀10、魔兽要诀5、召唤兽内丹5、储灵袋3、强化石3、净瓶玉露3、彩果4、随机红包2、符石卷轴1
  - 配置里还带有书铁150、宝石1~~2、怪物卡片1~~9、金丹1~~2、宝珠钨金珍珠1~~2这5个参数区间，但由于上面ITEM池里**没有**能触发它们的 key（装备书铁/宝石/怪物卡片/九转金丹等），这几项**实际永远不会掉落**
- **固定奖励**：经验=平均等级×随机(420~~530)×4；银子=(平均等级×160+15000)×1；+1除妖军功、+1活跃度；玩家有"任务13/501"进行中时2%概率额外获得"传说物品"；玩家等级15~~60且在地图等级范围内2%概率获得"心魔宝珠"

### 2. 夏日炎炎

- 场景：地图1092/1501（二选一），任务类型1184，战斗类型160001
- 每日次数上限：30次
- **怪物构成**：单一怪物"夏日炎炎"（模型：进阶雾中仙），等级随队伍在69~~175间浮动，数量随人数在3~~10只间浮动（配置见 `其他表/活动数据.csv` 战斗类型160001行，主怪/喽啰编号均为20239；技能：奔雷咒/地狱烈火/水漫金山/高级水，带夜战属性）
- **掉落**（HDPZ["夏日炎炎"]：经验倍率4，银子倍率2，爆率45）
  - ITEM权重池：金柳露10、高级召唤兽内丹10、高级魔兽要诀10、魔兽要诀5、召唤兽内丹5、储灵袋3、净瓶玉露3、强化石3、彩果4、特效宝珠1、符石卷轴1
  - 同样带有书铁150、宝石1~~2、怪物卡片1~~8、金丹1~~2、宝珠钨金珍珠1~~2这5个参数区间，但ITEM池里没有对应触发key，**同样永远不会掉落**
- **固定奖励**：经验=平均等级×随机(220~230)×4；银子=(平均等级×160+15000)×2；+1除妖军功、+1活跃度；额外记入"夏日炎炎"日常任务计数

## 六、"书铁"等材料为何几乎不掉落（2026-07-20 已修复）

> **更新**：已在全部118个明雷掉落配置（47个明雷菜单项展开后的全部星级/段位条目）的 ITEM 权重池中补上 `装备书铁=8` 和 `怪物卡片=8` 两个坑位，权重取"常见档"。修改的文件：
> - `全局bl.txt`（运行时缓存，立即对当前跑着的服务端生效）
> - `Script/数据中心/活动配置.lua`（Lua默认配置，仅覆盖其中原本就有定义的18项：地煞星级1~7、天罡星级1~6、生肖1~5星；其余明雷活动在该文件里本来就没有默认定义，只存在于运行时缓存里，见下方"重要发现"）
>
> 原本 地煞：星级1/2/3 的 ITEM 池里已有 `装备书铁`，权重为2（稀有档），本次已统一提升为8以保持一致；其余115项为全新添加。
>
> **重要发现**：`Script/数据中心/活动配置.lua` 顶部有 `if 文件不存在(全局bl.txt) then HDPZ={...} else HDPZ=读取(全局bl.txt) end` 的判断——由于 `全局bl.txt` 已存在，服务器实际启动时**完全不会执行**该文件里的 Lua 默认表，全部从 `全局bl.txt` 加载。而该 Lua 文件里只定义了决战大雁塔、地煞1~7星、天罡1~6星、生肖1~5星、中秋逐月这15项，其余32个明雷活动（捣乱年兽、夏日炎炎、天机策/云仙谣/善恶如来/群雄逐鹿/黑神话/灭世魔帝各9星级、武神坛5段位、长安保卫战5种怪、盘龙岛4档等）**只存在于 `全局bl.txt` 里，Lua源码中从未定义过**——推测是运营期间通过GM工具直接编辑并保存到缓存文件，与源码逐渐脱节。这意味着如果将来 `全局bl.txt` 被删除重置，这32个活动的掉落配置会连同这次改动一起丢失，只会保留这15项+决战大雁塔/中秋逐月的默认值。

结算函数（如 `自写活动:完成捣乱年兽`）的实际流程是**两层判定**：

结算函数（如 `自写活动:完成捣乱年兽`）的实际流程是**两层判定**：

```lua
玩家数据[id].角色:添加经验(...)   -- 无条件发放，与爆率无关
玩家数据[id].角色:添加银子(...)   -- 无条件发放
if 取随机数()<=HDPZ["活动名"].爆率 then      -- 第一层：是否触发本次随机产出
    local 名称,数量 = 糖果生成产出(糖果产出物品计算(HDPZ["活动名"].ITEM),"活动名")  -- 第二层：ITEM权重抽奖
    玩家数据[id].道具:给予超链接道具(id,名称,数量,...)
end
```

`书铁/宝石/怪物卡片/金丹/宝珠钨金珍珠` 这5个字段**不是可以直接掉落的道具**，它们只是给 `Script/数据中心/活动配置.lua` 里 `__Funfa` 表中几个特殊 key 用的参数（等级/数量区间），只有当 **ITEM 权重池里恰好配置了对应 key，并且这次随机产出恰好抽中它**，才会去读这些字段决定实际数量/等级：

| ITEM池里的key | 触发后实际发放的道具 | 关联字段 |
|---|---|---|
| 装备书铁 | 制造指南书(50%)或百炼精铁(50%)，等级取"书铁"区间 | 书铁 |
| 灵饰书铁 | 元灵晶石(55%)或灵饰指南书(45%)，与"书铁"区间无关 | 无 |
| 宝宝书铁 | 炼妖石(70%)或上古锻造图策(30%) | 无 |
| 宝石 | 随机宝石，数量取"宝石"区间 | 宝石 |
| 九转金丹 | 九转金丹，数量取"金丹"区间 | 金丹 |
| 怪物卡片 | 怪物卡片，数量取"怪物卡片"区间 | 怪物卡片 |
| 附魔宝珠/钨金/珍珠 | 对应道具，数量/等级取"宝珠钨金珍珠"区间 | 宝珠钨金珍珠 |

**结论：世界上根本没有一个叫"书铁"的道具**——它是策划内部对"装备打造材料"的字段俗称，真正掉进背包的是"制造指南书"或"百炼精铁"。而**"捣乱年兽""夏日炎炎"以及本文档第三节列出的绝大多数明雷活动，其 ITEM 权重池里压根没有配置"装备书铁/灵饰书铁/宝宝书铁"这几个 key**（可自行对照上面各活动的 ITEM 池确认），所以不管刷多少次、爆率多高，都**永远不可能**触发"书铁"类掉落，这不是概率低，是这些活动的奖励池里根本没有这个坑位。

目前查到唯一带"装备书铁/灵饰书铁"key的明雷活动只有 **地煞：星级1/2/3**（见3.6节），且该key在池内权重占比也很低（约总权重的3%~5%），还要先通过爆率（50%~65%）这一关，综合命中率约1.5%~3%。

同理，`怪物卡片`这个字段在全部明雷活动的HDPZ配置中**从未被任何活动的ITEM池引用过**，理论上无法通过任何明雷活动掉落——但这只是说"这条路径"是死的，"怪物卡片"这个道具本身在游戏里是完整存在、正常运作的系统（本质是"变身卡"：使用后变身对应怪物外形并获得其专属技能加成，具体种类/属性见 `Script/数据中心/变身卡.lua` 的`变身卡范围`/`变身卡数据`），它有别的真实生效的获取渠道：精灵怪"妙手空空"偷取（`Script/战斗处理类/战斗处理.lua`）、部分世界任务（`梦魇夜叉.lua`、`五行年兽任务.lua`）、商城/NPC兑换（5000点卡）、嘉年华抽奖、翻一翻小游戏等，只是不会从本文档列出的明雷活动里掉出来。`宝石`/`九转金丹`/`钨金`/`附魔宝珠`则在斥候海妖、盘龙岛系列、客服小二、十二生肖系列等少数明雷活动的ITEM池里有配置，是可能从这些活动掉落的。
