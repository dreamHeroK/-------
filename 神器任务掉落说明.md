# 神器任务（副本）掉落与奖励说明

> "神器任务"是一套按种族+星期几分派的组队副本系统（NPC"传令天兵"，长安城1001地图），共设计17个艺名档位，实际可用15个、1个已注册但打不开、1个是完全没接入的重复文件。本文档聚焦其**进入条件**、**奖励/掉落机制**、**怪物构成**，并单独列出代码核查中发现的死代码/缺陷，供后续调整参考。数据来自对 `Script/副本处理类/神器/` 全部16个文件、`Script/对话处理类/对话调用/1001.lua`、`Script/系统处理类/辅助内挂类.lua`、`Script/战斗处理类/战斗计算/战斗胜利.lua`、`Script/初始化脚本.lua` 等文件的逐行核查。

## 二、进入条件（15个正常副本完全一致）

- **等级**：`local DJ = 69`，逐个文件写死（如 `鸿蒙石之鉴.lua:27`），低于69级直接拒绝。
- **组队**：必须组队且由队长领取，队伍人数需 **≥5人**（如 `鸿蒙石之鉴.lua:29-38`）。
- **每日限制**：15个副本共用同一把日限锁 `副本数据.神器任务.完成[id]`（不是分副本各自计数），最后一关结算时对全队成员写入（如 `鸿蒙石之鉴.lua:267`）。
  - 该表在 `Script/初始化脚本.lua:566` 仅于**服务器启动时**初始化一次（`神器任务={进行={},完成={}}`），全仓库未搜到任何"每日0点清空 `副本数据.神器任务.完成`"的代码——这把"每日一次"的锁具体如何在不重启服务器的情况下重置，源码层面无法确认，需要结合实际运维行为核实。
- **进行中互斥**：身上已有对应任务类型（如1670）时不可重复领取（`鸿蒙石之鉴.lua:47-50`）。
- **任务时限**：队伍总任务72000秒（20小时）内需通关全部阶段；但每一阶段刷出的NPC本身只给1小时（3600秒）作战时限，超时未挑战会被清（如 `鸿蒙石之鉴.lua:60-70,88,119,142...`）。

### 难度星级与"神器难度"字段

- 玩家点击★~★★★★★后，`1001.lua:1987-1993` 会在**开启副本调用之后**，把 `难度=1~5` 写入队伍所有成员的 `玩家数据[id].角色.神器难度`，供后续结算阶段读取——手动走NPC对话这条路径是正确生效的。
- 但每个副本 `开启副本(id,nandu)` 函数签名里的 `nandu` 参数本身**从未被使用**（函数体内只有一句自己读自己写的空转代码 `local 难度=角色.神器难度; 角色.神器难度=难度`，如 `鸿蒙石之鉴.lua:40-41`），真正生效的难度写入是1001.lua结算后单独做的那个循环，与`开启副本`本身无关。

## 三、奖励/掉落机制——完全脱离HDPZ，写死在代码里

`Script/数据中心/活动配置.lua` 里**没有任何**神器任务或15个艺名的HDPZ条目（已核查），说明这套副本不走《怪物掉落和任务掉落.md》里描述的通用"糖果产出/ITEM权重池"系统，而是在每个副本文件的 `:完成奖励(id组,战斗类型)` 里**逐阶段手写数值**（每个副本5~9段几乎一模一样的重复代码块，没有共享函数）。

### 1. 经验/银子/积分公式（15个副本完全一致，每一阶段过关都发放，不止终关）

```lua
local jlxs
      if 神器难度==1 then jlxs=1
elseif 神器难度==2 then jlxs=1.5
elseif 神器难度==3 then jlxs=2
elseif 神器难度==4 then jlxs=2.5
elseif 神器难度==5 then jlxs=3
end                                    -- 注意：没有 else 分支，见五-4
经验 = 等级*取随机数(2650,4100) + 500000*jlxs
银子 = 等级*取随机数(110,120)   + 50000*jlxs
神器积分 = 10*jlxs
```
（示例：`鸿蒙石之鉴.lua:442-455`）

### 2. 道具爆率——12个按难度递增，3个写死20%（不一致，见五-6）

- **12个副本**：`爆率 = 神器难度*5+20`，即难度1~5对应 **25%/30%/35%/40%/45%**（如 `鸿蒙石之鉴.lua:449`、`千机锁之梏.lua:418`、`轩辕剑之陨.lua:557` 等）。
- **仙族·周一/二/三这3个副本**（清泽谱之惠、华光玉之伤、星斗盘之约）：直接写死 `取随机数()<=20`，**固定20%，完全不受神器难度影响**（`清泽谱之惠.lua:455`、`华光玉之伤.lua:447`、`星斗盘之约.lua:590`）。

### 3. 道具权重池——按 `1~100` 掷骰分档，两种"配方"

命中爆率后，再掷一次 `取随机数(1,100)` 决定具体道具，全部15个副本的分档区间完全一致，只是中段3~4个坑位的道具种类分两种配方：

| 掷骰区间 | "常规配方"（11个副本） | "变体配方"（4个副本：华光玉之伤/四神鼎之怨/噬魂齿之争/千机锁之梏） |
| --- | --- | --- |
| ≤10 | 特效宝珠×1 | 同左 |
| ≤15 | 四神石礼包×1 | 特殊兽诀·碎片 |
| ≤20 | 神兽涎 | 特殊兽诀·碎片（部分副本为神兽涎，逐一为准） |
| ≤25 | 高级魔兽要诀×1 | 同左 |
| ≤30 | 6级宝石礼包×1 | 特殊兽诀·碎片/仙露丸子（各副本不完全相同） |
| ≤40 | 神兜兜×随机(1~3) | 同左 |
| ≤50 | 仙露小丸子×1 | 同左 |
| ≤60 | 7级宝石礼包×1 | 仙露丸子/7级宝石礼包（各副本不完全相同） |
| >60 | 灵犀之屑×3 | 同左 |

（示例：常规配方见 `鸿蒙石之鉴.lua:459-486`、`清泽谱之惠.lua:458-485`；变体配方见 `华光玉之伤.lua:453-476`、`噬魂齿之争.lua:526-549`、`四神鼎之怨.lua:509-533`、`千机锁之梏.lua:431-455`。四个变体副本彼此的中段坑位也并非完全相同，具体以各自文件为准，本表只标出"跟常规配方不同"这一共性。）

发放全部走 `道具:给予超链接道具(...)`，会带全服"xt"频道广播（如"梦幻西游为高手玩家打造的副本，XX少侠在鸿蒙石之鉴中跌跌撞撞，竟然得到了珍贵的..."）。

### 4. 通关转盘抽奖

每个副本最终阶段通关后调用 `抽奖处理:大转盘抽奖(id)`（`Script/商店处理类/抽奖处理.lua:204起`，14格权重轮盘，权重范围1~1150，与HDPZ无关，全部副本共用同一个轮盘），**唯独轩辕剑之陨这一处被注释掉**（`轩辕剑之陨.lua:343`），其余14个正常触发。

### 5. 神器积分用途

- 累加函数：`角色处理类:添加神器积分(num)`（`Script/角色处理类/角色处理类.lua:3867-3874`）。
- 消耗渠道：商城"神器积分"分类可兑换 超级否定信仰碎片(500分)、装备强化石(3000分)（`Script/商店处理类/商城处理类.lua:784-798`）；1001.lua的"神器积分兑换"选项打开对应商品列表（`1001.lua:1977-1979`）。
- 战斗奖励模块 `Script/战斗处理类/战斗计算/战斗奖励.lua` 里也有独立的神器积分发放逻辑（29/175-177/246/392-394行），与本文档的副本结算是两条不同的发放路径。

## 五、代码缺陷/死代码汇总

1. **`魂灭噬魂齿.lua` 是完全的死文件**：未被 `ScriptInit.lua` require，class名`副本_噬魂齿之争`与任务类型1600都和 `噬魂齿之争.lua` 完全重复，结构逐行一致，没有任何代码引用它——纯粹是遗留的未清理副本。
2. **`黄金甲之谜.lua` 注册了但打不开**：唯一调用它的地方是1001.lua里全部被注释掉的备选分支（`-- 副本_黄金甲之谜:开启副本(...)`），辅助内挂类的"人"族分支也直接跳过它走泪痕碗之念。即便手动强开，其战斗类型用的是 `160140` 而非其它副本的 `170xxx` 段，`战斗处理类/战斗计算/战斗胜利.lua` 里神器任务的分发段只覆盖 `170000~170139`，160140完全没有对应分支——打完第一场战斗后进程无法推进。
3. **`开启副本(id,nandu)` 的 `nandu` 参数是死参数**：所有15个正常副本函数体内都只对`神器难度`做自读自写的空转赋值，真正生效的难度写入是1001.lua在`开启副本`调用**之后**单独对全队做的循环（1001.lua:1987-1993），与传入的`nandu`本身无关。
4. **自动内挂（辅助内挂类）的星级选择是摆设**：`辅助内挂类:接神器任务(数字id,难度)`（`辅助内挂类.lua:653-667`，注释自称"修复BUG版"）把`难度`原样传给`开启函数(id,难度)`，但因为第3条的原因，这个`难度`同样从未被写入`玩家数据[id].角色.神器难度`。也就是说挂机面板里勾选更高星级**不会**改变实际结算时用到的难度倍率/爆率，副本仍旧沿用玩家上一次手动跟NPC对话时留下的`神器难度`值（如果角色从未手动对话过，则为`nil`）。而`:完成奖励`里的`jlxs`分支没有`else`兜底，`神器难度`为`nil`时会导致`500000*jlxs`等算式对nil做算术运算而报错——这是纯挂机玩家的一个实际崩溃风险点。
5. **副本扫荡状态面板读错了表**：`Script/系统处理类/系统处理类.lua:1848-1870` 给每个艺名（含6个从未建成的幽灵艺名）单独起了一行，读取 `副本数据[艺名].完成[玩家id]`；但实际代码**只写入共用键** `副本数据.神器任务.完成[id]`，从未写过任何 `副本数据.清泽谱之惠.完成[id]` 这类分艺名的表。也就是说除了"神器任务"这一总行准确外，其余21行（15个正常+6个幽灵）**永远显示未完成**，无论玩家今天实际打没打。
6. **道具爆率的难度联动不一致**：12个副本按`难度*5+20`随难度递增，仙族周一/二/三这3个副本却写死固定20%，完全不吃`神器难度`加成——大概率是早期先写的这3个、后来引入`BL`变量时忘了回填，而不是有意为之的设计差异。
7. **轩辕剑之陨漏了通关转盘抽奖**：15个副本里唯独它在终关结算处把 `抽奖处理:大转盘抽奖(id)` 整行注释掉了（`轩辕剑之陨.lua:343`），其余14个都正常触发。
8. **战斗中的人数保护是无效的历史遗留判断**：每个阶段的战斗触发处都有一句 `取队伍人数(id)<1 ... "队伍人数低于3人"`（如 `鸿蒙石之鉴.lua:332,340,348...`），但`开启副本`早已要求进门时队伍≥5人，这个`<1`的判断在正常流程里永远不会触发，报错文案里的"3人"也和实际的5人门槛对不上——是死代码。
9. **无按BOSS分档的奖励**：`:完成奖励`是逐阶段整段复制的，同一副本内不同阶段（不同BOSS）拿到的经验/银子/掉落公式完全相同，只受玩家等级和神器难度影响——即这套系统没有"越到后面的BOSS掉落越好"的设计，纯粹靠难度星级和运气决定收益。

## 六、相关但非本文重点的邻近系统

- `Script/角色处理类/神器类.lua`（902行）是另一套**门派神器装备**系统（五行槽位、17门派各2个专属技能），与本文档的"神器任务副本"是两回事，只是掉落物"灵犀之屑"会被这套装备系统用作强化材料（`神器类.lua:66-73`）。
- `Script/战斗处理类/战斗计算/神器计算.lua`（31行）是上述门派神器装备的战斗技能效果实现，同样跟副本掉落无关。
- "灵犀之屑"在可读的 `Script/数据中心/物品数据.lua` 源码里搜不到独立定义，但该文件本身是先整体从Excel导表（`ReadExcel1("物品数据","")`）再在Lua里追加自定义条目，抽奖处理里也明确写了它是"强化装备所需的原料之一"，应视为存在于外部Excel表而非缺失道具。
- `Script/战斗处理类/怪物属性.lua:66786-66946` 一带存在大段**被注释掉的神器难度怪物强度缩放代码**（怪物经验减免/法伤减免/物伤减免等按难度分档的倍率），说明"难度越高怪物越强"这个设计曾经写过，后来整体注释停用——目前神器难度只影响奖励大小/爆率，不影响怪物强度，跟这段被删掉的实现有关。
