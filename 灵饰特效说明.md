# 灵饰特效说明

> 本文基于服务端源码实际逻辑整理，涉及函数名/文件路径均可在服务端代码中交叉核对。

**核心文件**：`Script\数据中心\物品数据.lua`（灵饰属性/PropertyData定义）、`Script\角色处理类\装备处理类.lua`（灵饰淬灵/星辉石）、`Script\任务处理类\活动调用\Task5.lua`（灵饰打造任务）、`Script\角色处理类\道具处理类.lua`（灵饰处理/洗练/特性道具）、`Script\角色处理类\角色处理类.lua`（穿戴结算/套装检测）

灵饰是独立于普通装备的第二套饰品系统，共 **4个槽位**：耳饰、佩饰、戒指、手镯（`物品数据.lua` `wp_4`=1~4）。各部位有60/80/100/120/140/160六档等级款式。角色数据 `self.灵饰={}`（`角色处理类.lua:153`），穿戴时对应 `self.灵饰[1..4]`。

灵饰身上有**两层完全独立**的加成体系：(A) 幻化属性（人人都有的主/副词条），(B) 特性（需后天镶嵌钟灵石/灵箓才有的技能类特效）。

---

## 一、幻化属性（主/副词条）

打造或礼包开出灵饰时必带，无需额外操作。

- `灵饰属性`（`物品数据.lua:18-25,44-45`）：按部位定义主属性池/副属性池。例如戒指主属性池={伤害,防御}，副属性池={固定伤害,法术伤害,伤害,封印命中等级,法术暴击等级,物理暴击等级,狂暴等级,穿刺等级,法术伤害结果,治疗能力,速度}。
- `灵饰主属性`（`物品数据.lua:48-68`）/`灵饰副属性`（`物品数据.lua:70-90`）：每个属性按等级（60/80/100/120/140/160）给出 `{a=下限,b=上限}` 数值区间，副词条区间通常小于主词条。
- 存储：`道具.幻化属性 = {基础={类型,数值,强化}, 附加={ {类型,数值,强化}, ... }}`（1条主属性+1~3条副属性）。
- 不分品阶/稀有度，只按等级（60~160六档）决定数值区间。

**部位速查**（对照现有《装备打造攻略》说法，主属性略有细分）：

| 部位 | 主属性池 |
|---|---|
| 戒指 | 伤害、防御 |
| 耳饰 | 法术伤害、法术防御 |
| 手镯 | 封印命中、抵抗封印 |
| 佩饰 | 速度 |

上古玉魄（阴/阳）另有专属词条表（`物品数据.lua:44-45`）。

## 二、特性（技能类特效）

需镶嵌"钟灵石"或使用"灵箓"才会出现，不是必然带的。

- 候选池 `灵饰随机特性`（`物品数据.lua:12`）共12种：血气方刚、通真达灵、心无旁骛、健步如飞、163、心源、回春之术、风雨不动、固若金汤、自我陶醉、气壮山河、锐不可当。
- 效果表 `PropertyData`（`物品数据.lua:101-371`），每种特性含：
  - 单件加成：`{部位={pro=属性名,rate=系数,status=1}}`，单件效果=`rate×特性等级`
  - 套装加成：`套装1`~`套装10`（4件同名特性时，按等级总和算出套装等级1~10生效）
- 存储：`道具.特性 = {技能=特性名, 等级=1~6, 套装=nil或1~10}`。特性等级1~6（由镶嵌材料决定），套装等级1~10（由四件套等级总和决定）。

## 三、生成与概率

### 1. 幻化属性（打造）
1. NPC对话"灵饰淬灵"：放入 **灵饰指南书 + 元灵晶石**（`装备处理类.lua:666-704`），生成打造任务（`Task5.lua:101`），需额外收集"强化石"（按等级20~40个不等，`Task5.lua:105-129`）
2. 任务完成回调 `灵饰处理`（`道具处理类.lua:13178-13211`）：
   - 主属性：池中随机1个类型，数值 `取随机数(a,b)`，**完全均匀随机**
   - 副属性：随机1~3条（`取随机数(1,3)`），池中随机类型+数值区间，若类型重复则继承已有数值
3. `dz灵饰处理`（`道具处理类.lua:13213-`，礼包/GM发放）与"灵饰自选"用的本地 `灵饰处理`（`道具处理类.lua:1126-1152`）逻辑相同，但**后者的随机行被注释掉，直接取数值上限b**，与打造随机结果不完全一致，写攻略/客服口径时需注意区分。

### 2. 特性生成
- **灵箓**（`道具处理类.lua:15092-15142`）：完全随机，12选1技能；等级由 `取随机数(1,200)` 映射：<100→1级，<120→2级，<140→3级，<160→4级，<180→5级，≤200→6级（约1级49.5%、2~6级各10%左右）
- **钟灵石/XX钟灵石**（`道具处理类.lua:15167-15190`）：直接把道具自带的技能+等级写入灵饰特性，**100%成功**，技能/等级由钟灵石道具本身决定
- **钟灵石道具生成**：给予时若不指定参数则12选1随机（`道具处理类.lua:9514-9516`）；抽奖系统产出的钟灵石为随机1~5级（`商店处理类\抽奖处理.lua:108-110`）
- **钟灵石合成升级**：同等级同技能2颗可合成升1级，成功率见 `装备处理类:取钟灵石合成几率`（`装备处理类.lua:3644`）
- **钟灵石降级**：`装备处理类.lua:1844-1862`
- **淬灵石**：把灵饰已有特性"吸取"为1颗钟灵石道具，随后清空灵饰特性（`道具处理类.lua:15144-15165`）

## 四、存储字段与战斗生效链路

```
道具.幻化属性 = { 基础={类型,数值,强化}, 附加={ {类型,数值,强化}, ... } }
道具.特性     = { 技能=特性名, 等级=1~6, 套装=nil或1~10 }
道具.灵饰     = true
道具.部位类型 = "戒指"/"手镯"/"耳饰"/"佩饰"
道具.幻化等级 = 0起，星辉石每次镶嵌+1
```
角色身上：`玩家数据[id].角色.灵饰={[1..4]=道具id}`。

生效链路：
1. 穿戴/卸下（`角色处理类:佩戴灵饰` `6444-6501`、`卸下灵饰` `6505-6571`）：幻化属性各条目累加进 `self.装备属性`；若有 `特性` 也按 `PropertyData` 单件系数累加
2. 汇总最终属性（`角色处理类.lua:4899-4917`）：遍历 `灵饰战斗属性`，装备+本命+阵营+玉魄属性相加
3. 战斗单位建立时同步（`战斗处理类:取灵饰属性`，`战斗处理.lua:2949-2963`）；穿刺等级、狂暴等级额外做 `/等级/30` 换算
4. 四件套判定（`角色处理类:检测钟灵石`，`角色处理类.lua:5166-5237`）：四个灵饰特性技能相同时，按等级总和算出套装等级1~10，写回 `特性.套装` 及 `玩家数据[id].角色.钟灵石套装`，触发点在穿戴/卸下灵饰后（`道具处理类.lua:16206`/`16214`）
5. 套装效果结算（`角色处理类.lua:4920-4949`）：气血/伤害/速度/防御/法防/抗暴/抵抗封印/固定伤害/气血回复/治疗能力/全属性等百分比或固定值加成；两个特殊即时触发效果在战斗流程内单独处理（"163"复活几率递减：`战斗处理.lua:13827`；"心源"每回合25%触发伤害加成、"163"气血恰好163时伤害加成：`战斗处理.lua:14247-14260`）

## 五、后天洗练/更换道具

| 道具 | 作用 | 位置 | 概率/限制 |
|---|---|---|---|
| 星辉石 | 强化幻化属性附加值，逐级镶嵌 | `装备处理类.lua:978-1056` | 无失败概率；需按1、2、3…等级顺序镶嵌，受"镶嵌限制等级=装备等级/10"及活动期特殊上限(15或16级)约束 |
| 灵饰洗练石 | 保持词条类型不变，仅重随数值 | `道具处理类.lua:14685-14709` | 70%成功率；已被星辉石强化过或160级灵饰不可用 |
| 灵饰属性石 | 重随副属性类型+数量（1~2条，50%概率强制3条） | `道具处理类.lua:14710-14729` | 70%成功率，与洗练石共用同一判定 |
| 灵箓 | 随机附加特性（12选1技能+1~6级） | `道具处理类.lua:15092-15142` | 见第三节等级分档；仅限已鉴定灵饰 |
| 钟灵石/XX钟灵石 | 指定技能+等级直接写入特性 | `道具处理类.lua:15167-15190` | 100%成功，替换旧特性 |
| 淬灵石 | 提取已有特性为钟灵石道具，灵饰特性清空 | `道具处理类.lua:15144-15165` | 无失败概率，前提是灵饰已有特性 |
| 灵饰图鉴（鉴定） | 鉴定未鉴定灵饰 | `道具处理类.lua:14876-14919` | 额外1.5%概率附赠"超级简易"特效（`特效`字段，与"特性"是不同字段） |
| 超简易宝珠 | 尝试给已鉴定灵饰附加"超级简易"特效 | `道具处理类.lua:15066-15090` | 21%成功率 |

星辉石/洗练石/属性石作用于(A)幻化属性层，灵箓/钟灵石/淬灵石作用于(B)特性(技能)层，两层互不干扰。
